How It Works

The Process

  1. Upload File — Drag & drop or click to select
  2. Compute Hash — SHA-256 fingerprint created
  3. Check VirusTotal — Query 70+ antivirus vendors
  4. Get Result — CLEAN, SUSPICIOUS, or MALICIOUS

Why This Works

File hashing is deterministic—the same file always produces the same hash. If 5+ antivirus engines have flagged that hash, it's probably malicious. If none have, it's likely safe.

We don't send the file itself to VirusTotal—just the hash. This keeps your files private.

Security & Privacy

Your data is important. Here's what we do and don't do.

✓ What We Do

  • Hash files with SHA-256
  • Check hashes against VirusTotal
  • Detect suspicious extensions
  • Keep local scan history
  • Delete files immediately

✗ What We Don't Do

  • Store files on servers
  • Read file contents
  • Detect zero-day exploits
  • Track your IP address
  • Replace your antivirus

Why This Project Exists

Most antivirus tools are black boxes. You upload a file, click "scan," and get a verdict without understanding how or why.

I wanted to build something transparent. Something that shows you exactly what's happening: your file gets hashed, that hash gets checked against 70+ security vendors, and you get an honest result with reasoning.

The code is on GitHub. You can see exactly how it works. No magic. No mystery. Just good security fundamentals.

What I Learned

  • APIs: Working with rate limits and error handling
  • Risk Classification: How extension-based detection actually works
  • Deployment: Getting Python apps production-ready
  • UI/UX: Security tools need clear, simple interfaces
